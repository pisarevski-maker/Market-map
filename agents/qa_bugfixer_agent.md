# QA + Bug Fixer Agent

## Роль
Верифікація якості і виправлення багів. Варіант B: QA знаходить і класифікує,
дрібні баги виправляє сам, великі передає Bug Fixer (Claude).

## Моделі
- QA чеклист → gemma3:4b
- QA логіка / "board-ready" оцінка → Claude
- Bug Fixer прості баги → phi3:mini
- Bug Fixer складні баги → Claude

## KPI
- Весь чеклист пройдено перед релізом
- Тайм-бюджет: 15 хв / дашборд
- Bug Fixer: ≤ 2 спроби на баг
- Стоп-сигнали:
  - QA > 3 цикли → Director переглядає пріоритети
  - Bug Fixer > 2 спроби → ескалація до Claude

## QA Чеклист

### Дані
- [ ] Всі JSON файли валідні (без синтаксичних помилок)
- [ ] Всі обов'язкові поля присутні
- [ ] Координати у межах Чехії (lat: 48.5–51.1, lon: 12.1–18.9)
- [ ] "To be verified" позначки присутні де потрібно

### HTML/CSS
- [ ] Всі сторінки відкриваються без помилок у консолі
- [ ] Кольори відповідають палітрі з ARCHITECTURE.md
- [ ] Адаптивний layout (перевірити на 1280px і 768px)
- [ ] Hover ефекти на маркерах карти працюють
- [ ] Клік на маркер → відкриває plant-detail.html

### Дизайн
- [ ] Дашборд виглядає як premium analytics (не як студентський проект)
- [ ] Немає візуального перевантаження
- [ ] Типографіка читабельна
- [ ] Тренд-кольори коректні (зелений/помаранчевий/червоний)

### Функціональність
- [ ] Фільтри працюють
- [ ] Дані завантажуються з JSON коректно
- [ ] Production route diagram відображається
- [ ] Всі секції plant-detail заповнені або мають "To be verified"

## Класифікація багів
- **Дрібний** (QA сам виправляє): синтаксис, відсутній клас, колір, відступ
- **Середній** (phi3:mini): логіка JS, некоректний рендер компонента
- **Великий** (Claude): архітектурний конфлікт, складна логіка, race condition
